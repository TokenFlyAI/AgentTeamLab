# Kalshi Alpha Dashboard — Product Spec

## Goal
Build a **live Kalshi trading dashboard** — a single-page web app that shows real-time trading signals, market edges, P&L, and strategy health. This is the core product the civilization is building together.

## Shared Codebase Location
ALL code goes in: `agents/bob/backend/`
- **API server**: `agents/bob/backend/dashboard_api.js` — Express server on port 3200
- **Frontend**: `agents/bob/backend/dashboard/index.html` — single-page dashboard
- **Signal data**: `agents/bob/output/trade_signals.json` — generated by live_runner.js

## Architecture

```
live_runner.js (grace runs every 10 min)
       ↓ writes
trade_signals.json + pnl_tracker output
       ↓ read by
dashboard_api.js (Express, port 3200)
       ↓ serves
dashboard/index.html (HTML/JS frontend)
```

## Screens / Features

### 1. Live Signals Panel
- List of current trade signals from trade_signals.json
- Each signal: market ticker, direction (YES/NO), confidence, position size, strategy name
- Color: green=BUY_YES, red=BUY_NO, grey=HOLD
- Auto-refresh every 60 seconds

### 2. Market Edge Scanner
- Read from agents/grace/output/econ_edges_today.json
- Show top 10 markets by edge magnitude
- Columns: market title, model probability, market price, edge %, recommendation

### 3. P&L Tracker
- Read from agents/bob/backend/strategies/output/pnl_tracker output
- Show: total paper P&L, win rate, number of trades, best/worst trade
- Simple running total chart (last 7 days if data available)

### 4. Strategy Health
- For each strategy (mean_reversion, momentum, crypto_edge, nfp_nowcast, econ_edge):
  - Last run time
  - Signal count
  - Status (OK / ERROR / NO_DATA)

### 5. Pipeline Run Button
- "Run Pipeline Now" button → calls POST /api/run-pipeline
- Server runs `node live_runner.js` and returns new signals

## API Endpoints (dashboard_api.js)
- GET /api/signals — returns trade_signals.json
- GET /api/edges — returns econ_edges_today.json
- GET /api/pnl — returns P&L summary
- GET /api/health — strategy health status
- POST /api/run-pipeline — triggers live_runner.js

## Definition of Done
The dashboard is DONE when:
1. `node agents/bob/backend/dashboard_api.js` starts without error
2. Opening `agents/bob/backend/dashboard/index.html` in a browser shows live signals
3. The "Run Pipeline" button works and updates signals
4. All 5 endpoints return valid JSON
