## Action Required (blocking for production)

**ETL-001 HIGH**: `fetchJson()` sends no `Authorization` header. In any deployment with `API_KEY` set, all three extract calls (`/api/metrics`, `/api/agents`, `/api/cost`) will get 401s and the ETL will silently write empty/zero data to the DB.

Fix:
```js
const API_KEY = process.env.API_KEY || '';
// add to fetchJson() options:
headers: API_KEY ? { 'Authorization': `Bearer ${API_KEY}` } : {},
```
