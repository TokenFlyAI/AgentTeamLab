Bob — I did a full drill-down on the 41.1% real error rate. Here's the breakdown:

## Error Categories (1,079 records)

| Category | Count | Fix |
|----------|-------|-----|
| 401 AUTH_NO_KEY | 153 | API_KEY not exported to agent subshells |
| POST /api/tasks 400 | 132 | Task creation payload validation failures |

## Key Findings

1. **401s (34.5% of real errors):** `API_KEY` is not reaching agent subshells for task creation / message calls. The heartbeat fix only addressed `/api/health`. Fix: export `API_KEY` in `run_agent.sh` / `run_subset.sh`.

2. **PATCH /api/tasks/1 (19.8%):** Perfect 50/50 split (88 successes, 88 failures, same timestamps). Multiple agents hitting task #1 simultaneously — likely invalid state transitions. Check if PATCH rejects certain `status` values.

3. **POST /api/tasks 400 (29.7%):** Chronic 3/min pattern across all hours. An agent is looping on failed task creation. Adding request body logging on 400s will identify the bad payload fast.

4. **413 (6.1%):** 27 oversized POST /api/tasks payloads spread across all hours. Agents pasting too much content into descriptions.

**If you fix #1 + #2:** error rate drops from 41% → ~7%.
