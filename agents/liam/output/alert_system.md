## Overview

The Alert Manager (`backend/alert_manager.js`) provides real-time notifications when trading signals fire with confidence >= 0.80. It polls the SignalEngine output and alerts via local JSON log and optional Slack webhook.

---

## Features

- **Real-time polling**: Checks for new signals every 5 seconds (well under 30s target)
- **Confidence filtering**: Only alerts on signals >= 0.80 confidence (per team consensus)
- **Duplicate prevention**: Tracks already-alerted signals to avoid spam
- **Dual alerting**: Logs to `alerts.json` + optional Slack webhook
- **Health endpoint**: Optional HTTP endpoint for monitoring the monitor
- **Graceful shutdown**: Handles SIGINT/SIGTERM cleanly

---

## Installation

The alert manager is located at:
```
backend/alert_manager.js
```

No additional dependencies required — uses only Node.js built-ins.

## Configuration

### Environment Variables

| Variable | Default | Description |
|----------|---------|-------------|
| `SIGNALS_FILE` | `../agents/bob/output/trade_signals.json` | Path to SignalEngine output |
| `ALERTS_FILE` | `../agents/liam/output/alerts.json` | Path to alert log |
| `SLACK_WEBHOOK_URL` | `null` | Slack incoming webhook URL |
| `PAPER_TRADING` | `true` | Flag included in alert payload |

### Example .env

```bash
# Optional: Custom paths
export SIGNALS_FILE="/custom/path/trade_signals.json"
export ALERTS_FILE="/custom/path/alerts.json"

# Optional: Slack notifications
export SLACK_WEBHOOK_URL="<your-slack-webhook-url>"

# Paper trading flag (default: true)
export PAPER_TRADING="true"
```

## Usage

### Start the Alert Manager

```bash
# Basic mode (polling only)
node backend/alert_manager.js

# With health endpoint on port 3250
node backend/alert_manager.js --health 3250
```

### Run in Background

```bash
# Using PM2
pm2 start backend/alert_manager.js --name alert-manager

# Or nohup
nohup node backend/alert_manager.js > /var/log/alert_manager.log 2>&1 &
```

## Alert Payload

When a signal fires with confidence >= 0.80, the following is logged:

```json
{
  "market": "BTCW-26-JUN30-100K",
  "signal_type": "mean_reversion",
  "confidence": 0.95,
  "direction": "yes",
  "timestamp": "2026-04-03T19:05:23.123Z",
  "paper_trading": true,
  "signal_source": "mock_fallback",
  "signal_generated_at": "2026-04-03T19:05:20.456Z",
  "expected_edge": 22,
  "recommended_contracts": 29,
  "reason": "Mean reversion: z-score=-6.94, mean=85.6, vol=890000"
}
```

### Slack Notification Format

When `SLACK_WEBHOOK_URL` is configured, alerts are posted as:

```
🟢 *Trading Signal Fired*

Market:           BTCW-26-JUN30-100K
Strategy:         mean_reversion
Direction:        YES
Confidence:       95%
Paper Trading:    Yes
Timestamp:        2026-04-03T19:05:23.123Z
```

## Troubleshooting

| Issue | Cause | Solution |
|-------|-------|----------|
| No alerts generated | Signals below 0.80 confidence | Check signal confidence in trade_signals.json |
| Slack not receiving | Missing SLACK_WEBHOOK_URL | Set env var with valid webhook URL |
| Stale alerts | Signals file not updating | Check live_runner is running |
| Duplicate alerts | Alert manager restarted | Expected — dedup cache is in-memory only |

## Runbook

### Alert Manager Down

1. Check process: `ps aux | grep alert_manager`
2. Check logs: `tail -f /var/log/alert_manager.log`
3. Restart: `node backend/alert_manager.js`
4. Verify health: `curl http://localhost:3250/health`

### Missing Alerts

1. Check signals file exists: `ls -la agents/bob/output/trade_signals.json`
2. Verify signal confidence >= 0.80: `cat trade_signals.json | jq '.signals[].confidence'`
3. Check alerts.json: `cat agents/liam/output/alerts.json`
4. Review logs for errors
