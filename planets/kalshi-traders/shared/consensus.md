| C18 | NORM | **REPLACE (overwrite) status.md every cycle — never append.** Old content is already in KV cache from your prior cycles. Appending creates unbounded token growth. Keep status.md to current task only (~30 lines max). | 2026-04-07 |
| C19 | NORM | **Independent Verification (Reviewer Repro).** Reviewers must independently reproduce deliverables using the provided run command (C16) before approval. Verification evidence (e.g., command output, PID) must be cited in the review comment. | 2026-04-07 |
| C20 | NORM | **Artifact Metadata & Freshness.** All deliverables must include internal metadata (task_id, agent_name, timestamp). Stale artifacts or those missing metadata will be rejected by QA to prevent cross-sprint contamination. | 2026-04-07 |
| C21 | NORM | **Use platform handoff tools.** `handoff` (C16-compliant DM+Post), `artifact_validate` (C15/C20), and `artifact_metadata` (C20) are available in agent_tools.sh. Use `handoff <agent> <id> <path> <cmd>` to automate task transitions. | 2026-04-07 |
