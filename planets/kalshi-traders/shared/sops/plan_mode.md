# Plan Mode SOP

## Overview

Plan mode is for deliberate, structured work: requirements gathering, architecture design, task decomposition. Bias toward documenting and aligning before building.

## Priorities

1. Handle new inbox messages (already shown in your context delta)
2. Document and clarify requirements for upcoming work
3. Create tasks for the next sprint — detailed, assignable, with clear acceptance criteria
4. Review architecture and flag risks or dependencies
5. Align with Alice on priorities before implementing

## Work Cycle

- New messages and tasks are delivered in your context delta — trust it, don't re-scan
- Handle new inbox messages before starting new work (they're already shown to you)
- Focus on planning deliverables: designs, specs, task breakdowns, risk assessments
- Write findings to `output/` and post milestones to team_channel
- Keep Alice informed of any blocking decisions

## Communication

- DM teammates to gather requirements or validate assumptions
- Post architecture decisions and plans to team_channel for visibility
- Escalate scope or priority conflicts to Alice

## Quality

- Plans must be concrete enough to execute without clarification
- Every task created must include: title, description, assignee, acceptance criteria
- Document assumptions explicitly — flag what is uncertain

## Culture

Consensus.md is pre-loaded in your context. Reference it from memory. Add entries for major architectural decisions (`type: decision`).
