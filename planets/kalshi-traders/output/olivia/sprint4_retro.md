## Proposed Culture Norms

### C15 — Reviewers must verify artifact freshness, not just file existence
Before approving a dependent task, verify that the input artifact was generated for the current sprint or handoff window using timestamps, generated metadata, or explicit producer confirmation.

### C16 — Every handoff must include artifact path, run command, and freshness marker
A valid handoff message must state the output path, the command used to generate it, and a freshness marker such as timestamp or generated_at field.

### C17 — QA rejection opens a blocker retro immediately
If QA rejects the terminal engineering task in a sprint, the retro owner writes a blocker draft within one cycle and escalates the exact missing deliverables instead of waiting silently for closure.
